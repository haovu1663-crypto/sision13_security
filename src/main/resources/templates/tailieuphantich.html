01 — Overview
Tổng Quan Hệ Thống
& Phân Quyền

Hệ thống quản lý khách sạn bao gồm 4 module chính: Quản lý nhân viên, Quản lý phòng, Quản lý tài khoản người dùng và Quản lý đặt phòng/trả phòng. Hệ thống hỗ trợ đặt phòng qua 2 kênh và 2 phương thức thanh toán.

Phân quyền hệ thống (Roles)

ADMIN

  • Toàn quyền hệ thống
  • CRUD nhân viên, phân quyền
  • Xem báo cáo doanh thu
  • Cấu hình hệ thống
  • Quản lý tài khoản user

MANAGER

  • Quản lý phòng (CRUD)
  • Xem danh sách nhân viên
  • Xem & duyệt đặt phòng
  • Xem báo cáo doanh thu
  • Quản lý loại phòng & giá

RECEPTIONIST

  • Đặt phòng trực tiếp
  • Check-in / Check-out
  • Xử lý thanh toán
  • Xem trạng thái phòng
  • Cập nhật thông tin khách

CUSTOMER

  • Đặt phòng online
  • Xem lịch sử đặt phòng
  • Huỷ đặt phòng
  • Cập nhật hồ sơ cá nhân
  • Thanh toán cọc online
02 — Database
Thiết Kế Entity
& Database Schema
👤

User

  • id, username, password (BCrypt)
  • fullName, email, phone
  • role: ADMIN / MANAGER / RECEPTIONIST / CUSTOMER
  • status: ACTIVE / LOCKED
  • createdAt, updatedAt
🏢

Employee

  • id, employeeCode
  • user (OneToOne → User)
  • department, position
  • hireDate, salary
  • status: WORKING / RESIGNED / ON_LEAVE
🛏️

Room

  • id, roomNumber, floor
  • roomType (ManyToOne)
  • status: AVAILABLE / OCCUPIED / CLEANING / MAINTENANCE
  • description, images
🏷️

RoomType

  • id, name (SINGLE / DOUBLE / SUITE / ...)
  • pricePerNight, capacity
  • amenities (JSON hoặc OneToMany)
  • description
📋

Booking

  • id, bookingCode
  • customer (ManyToOne → User)
  • room (ManyToOne → Room)
  • checkInDate, checkOutDate
  • bookingType: ONLINE / DIRECT
  • status: PENDING / CONFIRMED / CHECKED_IN / CHECKED_OUT / CANCELLED
  • totalAmount, depositAmount
  • createdBy (nhân viên nếu DIRECT)
💳

Payment

  • id, booking (ManyToOne)
  • amount, paymentDate
  • method: CASH / BANK_TRANSFER
  • type: DEPOSIT / FULL_PAYMENT / REFUND
  • status: PENDING / SUCCESS / FAILED
  • transactionCode, bankRef
  • processedBy (nhân viên)
JAVA / JPA
@Entity
@Table(name = "bookings")
public class Booking {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(unique = true, nullable = false)
    private String bookingCode;  // BK-20240422-0001

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "customer_id")
    private User customer;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "room_id")
    private Room room;

    private LocalDate checkInDate;
    private LocalDate checkOutDate;

    @Enumerated(EnumType.STRING)
    private BookingType bookingType;  // ONLINE | DIRECT

    @Enumerated(EnumType.STRING)
    private BookingStatus status;

    private BigDecimal totalAmount;
    private BigDecimal depositAmount;   // 30% khi đặt online

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "created_by")
    private User createdBy;  // null nếu khách tự đặt online

    @OneToMany(mappedBy = "booking", cascade = CascadeType.ALL)
    private List<Payment> payments;

    @CreationTimestamp
    private LocalDateTime createdAt;
}

05 — Payment
Xử Lý Thanh Toán
Tiền Mặt & Chuyển Khoản
💵

Tiền Mặt (CASH)

Nhân viên thu tiền trực tiếp tại quầy, xác nhận số tiền nhận được, hệ thống ghi nhận payment ngay lập tức.

  • Không cần transaction code
  • Nhân viên là người xác nhận (processedBy)
  • Cập nhật status → SUCCESS ngay
  • In hoá đơn (PDF)
🏦

Chuyển Khoản (BANK_TRANSFER)

Khách chuyển khoản qua app ngân hàng / cổng thanh toán. Hệ thống lưu mã tham chiếu và xác nhận khi nhận được tiền.

  • Lưu transactionCode từ ngân hàng
  • Tích hợp webhook hoặc xác nhận thủ công
  • Trạng thái: PENDING → SUCCESS/FAILED
  • Lưu bankRef để đối soát

Logic thanh toán khi Check-out: Tổng hóa đơn = pricePerNight × số đêm. Nếu đặt online và đã cọc, số tiền còn lại = totalAmount − depositAmount. Nếu đặt trực tiếp, thanh toán toàn bộ khi trả phòng. Hệ thống tự động tính và hiển thị số tiền cần thu.

JAVA — PaymentService
@Service
@Transactional
public class PaymentService {

    public PaymentResponse processCheckoutPayment(Long bookingId, PaymentRequest req) {

        Booking booking = bookingRepo.findById(bookingId)
            .orElseThrow(() -> new BookingNotFoundException(bookingId));

        // Tính số tiền còn lại cần thu
        BigDecimal alreadyPaid = paymentRepo.sumSuccessfulPayments(bookingId);
        BigDecimal remaining = booking.getTotalAmount().subtract(alreadyPaid);

        if (req.getAmount().compareTo(remaining) < 0) {
            throw new InsufficientPaymentException("Số tiền không đủ");
        }

        Payment payment = new Payment();
        payment.setBooking(booking);
        payment.setAmount(req.getAmount());
        payment.setMethod(req.getMethod());  // CASH | BANK_TRANSFER
        payment.setType(PaymentType.FULL_PAYMENT);
        payment.setProcessedBy(getCurrentUser());

        if (req.getMethod() == PaymentMethod.CASH) {
            payment.setStatus(PaymentStatus.SUCCESS);
        } else {
            payment.setTransactionCode(req.getTransactionCode());
            payment.setBankRef(req.getBankRef());
            payment.setStatus(PaymentStatus.PENDING); // chờ xác nhận bank
        }

        paymentRepo.save(payment);

        if (payment.getStatus() == PaymentStatus.SUCCESS) {
            booking.setStatus(BookingStatus.CHECKED_OUT);
            booking.getRoom().setStatus(RoomStatus.CLEANING);
        }

        return mapToResponse(payment);
    }
}
